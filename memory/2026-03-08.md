- Discord channel policies differ from WhatsApp

- 'groupPolicy' options: 'open'|'allowlist'|'disabled'


## Exec Approvals Configuration Fix

### Problem
- Discord approval buttons not appearing for exec commands
- Gateway rejecting config with "Unrecognized key: 'interactive'"
- approvers placed in wrong location

### Root Cause
- `gateway.interactive` is not a valid config path in OpenClaw v2026.3.2
- `channels.discord.approvers` must be nested under `execApprovals`
- execApprovals.enabled defaults to false when block absent

### Correct Configuration
```json
"channels": {
  "discord": {
    "enabled": true,
    "token": "...",
    "dmPolicy": "open",
    "allowFrom": ["*"],
    "groupPolicy": "open",
    "execApprovals": {
      "enabled": true,
      "approvers": ["1037535689992061049"]
      // target: "dm" (default) or "channel" or "both"
    }
  }
}
```

### Resolution Steps (2026-03-08 18:37-18:42 PDT)
1. Removed invalid `gateway.interactive: true`
2. Moved approvers into `discord.execApprovals.approvers`
3. Restarted gateway: `openclaw gateway restart`
4. Verified: gateway active with new PID (176870)
