## 🏢 **新项目启动 - TWE_CRM 系统（2026-03-09 16:00）**

### **需求来源**
公司保险业务部门需要一套安全的CRM系统，用于：
- 统一管理客户信息与保单
- 整合 Gmail 邮件与 Dropbox 文件
- 支持工作流任务跟踪与续保提醒
- 提供 SOP 流程引导新员工

### **技术选型（快速确认）**
- **后端**: NestJS + PostgreSQL
- **前端**: Next.js 14 (App Router) + TailwindCSS
- **认证**: JWT + Google OAuth (Gmail API)
- **文件存储**: Dropbox API
- **部署**: Docker Compose (单服务器)
- **成本**: $5-10/月 (Linode/VPS)

### **架构设计要点**
- **数据模型**: 客户 multi-policy，保单独立表，多对多关联
- **权限**: RBAC (admin/manager/agent)
- **邮件同步**: Gmail API → 自动关联客户 → 附件存 Dropbox
- **续保流程**: 每日扫描 → 创建 RenewalReminder → 生成 Task → Kanban 看板
- **SOP**: 动态表单引导，按产品类型（life/auto/home）显示步骤

### **核心模块（已完成 scaffold）**
1. **Auth** – Google OAuth 登录，JWT 签发
2. **Users** – 员工管理（角色：admin/manager/agent）
3. **Customers** – 客户 CRUD + 保单关联
4. **Policies** – 保单管理（type, insurer, dates, fileUrl, metadata）
5. **Tasks** – 工作流任务（status, priority, dueDate）
6. **Renewals** – 续保提醒自动生成
7. **ActivityLogs** – 操作审计
8. **SOP** – 模板与表单配置

### **API 设计（RESTful）**
- `POST /auth/google` → OAuth 跳转
- `GET /customers` → 列表（按角色过滤）
- `POST /policies` → 创建保单
- `GET /tasks` → 任务列表（Kanban）
- `POST /renewals/scan` → 手动触发续保扫描
- `GET /sop/form/:type` → 获取 SOP 表单配置

## ☁️ **Render 部署实战 - TWE_CRM（2026-03-09 19:00-20:17）**

### **问题背景**
目标：将 TWE_CRM 部署到 Render 免费 tiers（PostgreSQL + Redis + Web Services）
挑战：构建阶段连续失败，需要解决依赖、路径、配置问题

### **遇到的错误与解决方案**

#### **1. 构建错误：`npm ci` 缺少 package-lock.json**
```
npm error The npm ci command can only install with an existing package-lock.json...
```
**解决**：修改 `render.yaml` 的 `buildCommand` 从 `npm ci --only=production` 改为 `npm install`
- 允许安装 devDependencies（需要 @nestjs/cli 等开发工具）
- 提交：`ae63763` → `076d282`

#### **2. 构建错误：`@swagger-nodegen/cli` 不在 npm registry**
```
404 Not Found - GET https://registry.npmjs.org/@swagger-nodegen%2fcli
```
**解决**：从 `backend/package.json` 的 `devDependencies` 中移除该废弃包
- 该包仅用于生成 Swagger 客户端代码，生产环境不需要
- 提交：`076d282` → `fc7322e`

#### **3. 前端构建错误：缺少 shadcn/ui 组件**
```
Module not found: Can't resolve '@/components/ui/button'
```
**解决**：
- 添加 shadcn/ui 依赖（class-variance-authority, tailwindcss-animate）
- 创建 `components.json` 配置
- 创建 Minimal 组件：`button.tsx`, `input.tsx`, `textarea.tsx`, `select.tsx`
- 创建 `lib/utils.ts`（cn 函数）
- 修改所有页面，将复杂 Select 用法简化为 `<Select><SelectItem>...</SelectItem></Select>`
- 提交：`fc7322e` → `af0d2dd`

#### **4. Next.js 路径别名 `@` 未配置**
TypeScript `tsconfig.json` 定义了 `@/*`，但 webpack 需要额外配置。

**解决**：
- 创建 `frontend/tsconfig.json`（完整配置，不依赖 extends）
- 创建 `frontend/next.config.js` 配置 webpack 别名
- 添加 `frontend/next-env.d.ts` Next.js 类型声明
- 提交：`af0d2dd` → `5fd0842` → `a624df0` → `0b6193f` → `df4cb42` → `a9eeadb`

#### **5. TypeScript 类型包缺失**
```
It looks like you're trying to use TypeScript but do not have the required package(s) installed.
```
**解决**：将 `typescript`, `@types/react`, `@types/react-dom`, `@types/node` 从 `devDependencies` 移到 `dependencies`
- Next.js 生产构建需要这些类型包
- 提交：`df4cb42`

#### **6. `Link` 组件未导入**
```
Type error: Cannot find name 'Link'.
```
**解决**：批量修复所有使用 `<Link>` 的页面组件，添加 `import Link from 'next/link'`
- 修改文件：
  - `beneficiaries/[id]/edit/page.tsx`
  - `policies/new/page.tsx`
  - `policies/[id]/edit/page.tsx`
  - `beneficiaries/new/page.tsx`
  - `tasks/[id]/edit/page.tsx`
- 提交：`b231f9e`

#### **7. Backend 构建：`nest` 命令找不到**
```
sh: 1: nest: not found
npm error could not determine executable to run
```
**原因**：`@nestjs/cli` 在 `devDependencies` 中，`npm install --only=production` 不会安装它。

**解决**：
- 方案 A：`render.yaml` 使用 `npx nest build`（不依赖全局安装）
- 方案 B：将 `@nestjs/cli` 移到 `dependencies`（最终采用）
- 同时修改 `package.json` 的 `build`, `start`, `start:dev` 脚本都使用 `npx`
- 提交：`b231f9e` → `c56c38e`

#### **8. TypeScript 配置：`next/tsconfig.json` not found**
```
FatalError: error TS6053: File 'next/tsconfig.json' not found.
```
**解决**：创建完整的 `tsconfig.json` 不依赖 Next.js 的 extends，并添加 `next-env.d.ts`
- 提交：`c56c38e` → `a9eeadb`
